# Telegram Wikimedia Commons Bot

Unofficial Telegram bot and CLI for searching and reading Wikimedia Commons media.

The code in this repository was generated by Codex gpt-5.5 xhigh and reviewed/edited as a normal Rust project.

## Features

- Telegram search with up to 20 file buttons and a second message with up to 20 category buttons.
- Telegram inline mode, so users can type `@your_bot Minsk` and see Commons image previews before sending.
- Telegram inline location support: when the user shares inline location data, inline results show nearby geotagged Commons images; otherwise inline mode uses the typed search query.
- File callback sends the original file with metadata: license link, uploader link, upload date, original Commons link, optional geolocation link, upload version count, animation flag, and optional SHA-1.
- Category callback sends category description, Wikidata link and clickable Wikidata key/value claims, then direct files, then subcategories.
- Preferences in DynamoDB, with RAM cache in warm Lambda instances.
- Stateless mode via `STATELESS_MODE=true`, disabling DynamoDB and persistent preferences.
- CLI mode using the same parser and Commons client.
- CLI parseable output: `--format json`, `--format jsonl`, `--format tsv`.
- CLI image preview through Kitty `icat` when available and audio playback through `mpv`.
- CLI category download with optional subcategory recursion.
- Terraform deployment to AWS Lambda on `arm64`.
- Scripts for deploy, code update, logs, stats, and release tagging.
- GitHub Actions CI/CD with Rust checks, Terraform formatting, Sonar, optional live Lambda integration tests, and binary release uploads.

## Search Syntax

Examples:

```text
Minsk
-img Minsk
-links Minsk
image Minsk
bild Berlin
audio:bird
flac Minsk c birds
User:Vitaly_Zdanevich date:2025 Minsk
user:Vitaly_Zdanevich d:7days audio:something
s:>10MB s:<20MB Minsk
Category Minsk
Kategorie Berlin
Категория Минск
Катэгорыя Мінск
```

Aliases:

- Category: `category`, `c`, `Kategorie`, `kat`, `k`, `категория`, `катэгорыя`, `к`, `с`
- Images: `image`, `images`, `img`, `i`, `Bild`, `Bilder`, `Foto`, `Fotos`, `выява`, `ваява`, `в`
- Audio: `audio`, `sound`, `music`, `Ton`, `Klang`, `Musik`, `аудио`, `музыка`, `звук`, `аудыё`
- User: `user`, `u`, `User`, `Benutzer`
- Date: `date`, `d`, `Datum`
- Size: `size`, `s`, `Größe`, `Groesse`, `g`

Telegram max bot file size is 50 MB, so the bot filters out bigger files by default. Files above 20 MB use red buttons because the bot must download and upload them itself. Audio buttons use blue styling where Telegram clients support button styles.

Use `-img` to get one Telegram media group with up to 10 thumbnail previews and metadata captions. The bot uses 1280px Commons thumbnails for safer Telegram delivery and does not send extra overflow metadata messages for explicit `-img`. Use `-links` to get 10 compact Commons links in one message.

Inline mode works without location sharing. If Telegram sends inline location data, nearby geotagged images are ranked by distance and the typed text still filters file titles. Structured filters such as `user:`, category, date, size, extension, or audio/video type keep the normal Commons search path.

## Preferences

`/settings`, `/prefs`, and `/preferences` show the same preferences screen. Telegram users can use inline buttons with check marks for current values, or keep using command syntax:

```text
/settings mode buttons|links10|images10|images20
/settings type all|images|audio|video
/settings ext jpg|webp|flac|pdf|off
/settings category-counts on|off
/settings preview-metadata on|off
/settings sha1 on|off
/settings filesize on|off
/settings favorite add Category name
/settings blacklist-category add Category name
/settings blacklist-user add Username
```

The extension submenu is built from Wikimedia Commons' supported file extension list: `djvu`, `flac`, `gif`, `jpeg`, `jpg`, `mid`, `midi`, `mp3`, `mpeg`, `mpg`, `oga`, `ogg`, `ogv`, `opus`, `pdf`, `png`, `stl`, `svg`, `tif`, `tiff`, `wav`, `webm`, `webp`, `xcf`.

## CLI

```bash
cargo run --bin commons-cli -- "image Minsk"
cargo run --bin commons-cli -- --format jsonl --no-animation "flac Minsk c music"
cargo run --bin commons-cli -- --preview "jpg Minsk"
cargo run --bin commons-cli -- --play-audio "audio birds"
cargo run --bin commons-cli -- --download-category "Minsk" --recursive --output-dir ./downloads
```

`fzf` integration is intentionally script-friendly rather than built-in:

```bash
cargo run --bin commons-cli -- --format tsv --no-animation "jpg Minsk" | fzf --with-nth=3,4
```

Some `fzf` preview workflows can show images by calling `kitty +kitten icat`, `chafa`, or another terminal image tool from `--preview`; the CLI also has `--preview` for Kitty terminals.

## Deploy

Create `infra/terraform.tfvars` from the example:

```hcl
telegram_bot_token      = "123456:replace-me"
telegram_webhook_secret = "replace-with-random-secret"
admin_telegram_user_ids = "123456789"
enable_test_endpoint    = true
```

Deploy:

```bash
./scripts/deploy.sh
```

Update only code:

```bash
./scripts/update-code.sh
```

Upload optional Pywikibot cookies for authenticated Commons API requests:

```bash
./scripts/upload-commons-cookie.sh /home/vitaly/.pywikibot/pywikibot-Vitaly_Zdanevich.lwp
```

Store only the SSM parameter name in `infra/terraform.tfvars`:

```hcl
commons_auth_cookie_ssm_parameter = "/telegram-wikimedia-commons-bot/commons-auth-cookie-jar"
```

Show logs:

```bash
./scripts/show-logs.sh --since 2h
./scripts/show-logs.sh --since 24h --errors
```

Show stats:

```bash
./scripts/stats.sh
```

Live integration tests call the authenticated `GET /__test?q=...` Lambda Function URL path. Keep `enable_test_endpoint = true` when you want GitHub Actions to verify the deployed endpoint, and set the `LIVE_LAMBDA_URL` and `TELEGRAM_WEBHOOK_SECRET` repository secrets.

Release without deploying:

```bash
./scripts/release.sh -m "Release 0.1.1"
./scripts/release.sh --middle -m "Release 0.2.0" -m "Add larger feature set."
```

## AWS Cost Notes

The Terraform default uses `arm64` and 3008 MB, the largest Lambda memory size accepted by the AWS account used during initial deployment. That gives more CPU than small Lambdas, but it burns through the Lambda free GB-second allowance faster. Lower `lambda_memory_size` in Terraform if cost is more important than callback speed.

AWS free-tier references:

- Lambda pricing and free tier: https://aws.amazon.com/lambda/pricing/
- DynamoDB pricing and 25 GB free-tier storage: https://aws.amazon.com/dynamodb/pricing/
- Systems Manager Parameter Store standard parameters: https://aws.amazon.com/systems-manager/pricing/

The default region is `us-east-1`, close to Wikimedia's Ashburn application datacenter. Wikimedia data center reference: https://wikitech.wikimedia.org/wiki/Data_centers

Secrets Manager is intentionally not used for Wikimedia cookies because it has per-secret monthly pricing. Standard SSM Parameter Store parameters have no additional charge at standard throughput, and the Lambda caches the decrypted cookie header in RAM after the first read.

## Wikimedia Notes

Read-only Commons usage does not require Wikimedia credentials. The bot sets a descriptive User-Agent as recommended by MediaWiki API etiquette.

This project uses the MediaWiki Action API first. HTML parsing should only be used later for gaps that the API cannot expose cleanly.

Wikimedia Commons upload links shown in `/help`:

- Upload Wizard: https://commons.wikimedia.org/wiki/Special:UploadWizard
- Upload tools: https://commons.wikimedia.org/wiki/Commons:Upload_tools

All files on Wikimedia Commons are public.
